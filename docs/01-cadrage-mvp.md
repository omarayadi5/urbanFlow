# Cadrage MVP

## Objectif

Construire une application web/PWA de mobilite urbaine presentable pour un projet Titre 6 CDSD : une base technique reelle, separant frontend, backend, donnees et documentation.

## Perimetre initial

- Authentification : inscription, connexion, deconnexion, recuperation session, profil.
- Routes protegees : carte, dashboard, profil, parametres.
- Backend API : `auth`, `profile`, `transport`, `routing`, `demo`.
- Base de donnees : PostgreSQL via Supabase.
- Deploiement cible : Vercel pour le front, Render pour le back.

## Hors perimetre MVP

- Donnees temps reel transport officielles.
- Paiement, messagerie, role administrateur avance.
- Algorithme de routing complet type OpenTripPlanner.

## Contraintes soutenance

Le PDF de soutenance rappelle que le Titre 6 CDSD porte sur la conception et la realisation technique d'un dispositif digital. Le livrable doit demontrer les choix techniques, l'architecture, la securite, le developpement et les limites.
